# Mini Site Builder — TL:TECH

Мини-конструктор сайтов с админ-панелью на примере главной страницы портала [travelline.tech](https://travelline.tech).

Контент публичной страницы управляется через админку без ручной правки HTML и без перезапуска сервера.

Учебный проект летней практики.

## Возможности

- Публичная страница — копия главной `travelline.tech`.
- Серверный рендеринг через `EJS`.
- Админ-панель для управления 5 блоками:
- `Hero` — заголовок, подзаголовок, статистика.
- `Команда` — карточки сотрудников и загрузка фото.
- `Клиенты` — логотипы и названия клиентов.
- `Вакансии` — название, формат работы, ссылка.
- `Плюшки` — заголовок и описание бонуса.
- Данные хранятся в `data/data.json`.
- Автоматические тесты проверяют логику работы с данными.

## Стек

| Технология | Зачем используется |
|---|---|
| Node.js | среда выполнения JavaScript |
| Express | веб-сервер, API, статические файлы |
| EJS | серверный рендеринг HTML |
| Multer | загрузка фото сотрудников |
| JSON | хранение данных сайта |
| node:test | автоматические тесты |

## Запуск

Все команды выполняются из папки `project/`.

```bash
npm install
npm start
```

После запуска сервер будет доступен по адресу:

```text
http://localhost:3000
```

Админка:

```text
http://localhost:3000/admin.html
```

Остановить сервер можно сочетанием `Ctrl+C` в терминале.

## Как Пользоваться Админкой

1. Открыть `http://localhost:3000/admin.html`.
2. Изменить нужный блок.
3. Нажать кнопку сохранения внутри этого блока.
4. Открыть или обновить главную страницу `http://localhost:3000`.
5. Проверить, что изменения появились на публичной странице.

Если браузер показывает старые данные, используйте жёсткое обновление: `Ctrl+Shift+R`.

## Рендеринг

В проекте используется серверный рендеринг.

Схема работы публичной страницы:

```text
Браузер открывает /
Express читает data/data.json
Express передаёт данные в views/index.ejs
EJS формирует готовый HTML
Браузер получает готовую страницу
```

Схема работы админки:

```text
Админка -> POST /api/data -> data.json -> серверный рендер главной страницы
```

## API

| Метод | Адрес | Назначение |
|---|---|---|
| `GET` | `/` | публичная страница |
| `GET` | `/api/data` | получить все данные в JSON |
| `POST` | `/api/data` | сохранить данные в `data.json` |
| `POST` | `/api/upload/team-photo` | загрузить фото сотрудника |

## Структура Проекта

```text
project/
  src/
    server.js          Express-сервер, API, EJS-рендеринг
    storage.js         функции для работы с данными
  views/
    index.ejs          шаблон публичной страницы
  data/
    data.json          данные управляемых блоков
  tests/
    storage.test.js    автоматические тесты
  public/
    admin.html         админ-панель
    css/               стили админки и публичной страницы
    js/                JS-файлы админки и слайдеров
    img/               изображения сотрудников и клиентов
    site-assets/       файлы оригинального сайта
  reserve/             резервные/черновые файлы

```

## Основные Файлы

- `src/server.js` — запуск сервера, API, раздача статики, рендер главной страницы.
- `src/storage.js` — функции добавления, удаления и обновления данных.
- `views/index.ejs` — публичная страница, куда сервер подставляет данные.
- `data/data.json` — основной источник данных сайта.
- `public/admin.html` — интерфейс админ-панели.
- `public/js/admin-*.js` — логика редактирования отдельных блоков.

## Тесты

Запуск тестов:

```bash
npm test
```

Тесты находятся в `tests/storage.test.js` и проверяют:

- добавление элементов;
- удаление элементов;
- валидацию пустых полей;
- работу функций для команды, клиентов, вакансий и плюшек;
- чтение и запись JSON-хранилища.
