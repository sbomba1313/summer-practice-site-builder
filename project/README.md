# Mini Site Builder — TL:TECH

Мини-конструктор сайтов с админ-панелью на примере главной страницы портала
[travelline.tech](https://travelline.tech). Контент публичной страницы
управляется через админку без правки кода и перезапуска сервера.

Учебный проект летней практики.

## Возможности

- Публичная страница — копия главной travelline.tech (серверный рендеринг).
- Админ-панель для управления **5 блоками**:
  - **Hero** — заголовок, подзаголовок, статистика;
  - **Команда** — карточки сотрудников (+ загрузка фото с компьютера);
  - **Клиенты** — логотипы;
  - **Вакансии** — название, формат, ссылка;
  - **Плюшки** — заголовок и описание.
- Данные хранятся в одном JSON-файле (`data/data.json`).
- Изменения из админки сразу видны на публичной странице.
- Автоматические тесты логики работы с данными.

## Стек технологий

| Технология | Зачем |
|---|---|
| Node.js | среда выполнения |
| Express | веб-сервер, маршруты, статика |
| EJS | серверный рендеринг (подстановка данных в шаблон) |
| Multer | загрузка фото сотрудников |
| JSON-файл | хранилище данных |
| node:test | тесты (встроены в Node, без зависимостей) |

## Требования

- [Node.js](https://nodejs.org) (LTS) — проверка: `node --version`

## Запуск

Все команды выполняются из папки `project/`.

```bash
# 1. Установить зависимости (создаст node_modules)
npm install

# 2. Запустить сервер
npm start
```

После запуска в терминале появится `Сервер запущен на http://localhost:3000`.

- Публичная страница: <http://localhost:3000/>
- Админ-панель: <http://localhost:3000/admin.html>

Остановить сервер — `Ctrl+C`.

> После изменений в админке на публичной странице нажмите **Ctrl+Shift+R**
> (жёсткая перезагрузка), чтобы увидеть свежие данные.

## Тесты

```bash
npm test
```

Тесты (`tests/storage.test.js`) проверяют логику работы с данными из
`src/storage.js`: добавление, удаление и изменение элементов каждого блока,
валидацию (пустые поля → ошибка), а также чтение и запись хранилища.

## API

| Метод | Адрес | Назначение |
|---|---|---|
| `GET` | `/` | публичная страница (рендер из data.json) |
| `GET` | `/api/data` | вернуть все данные в JSON |
| `POST` | `/api/data` | сохранить присланные данные в data.json |
| `POST` | `/api/upload/team-photo` | загрузить фото сотрудника, вернуть путь |

## Как это работает

```
Админка (формы)  ──POST /api/data──►  data.json  ──читает сервер──►  публичная страница
   admin-*.js                        (источник)        EJS-рендер         index.ejs
```

Админка и публичная страница напрямую не связаны — обмен идёт через `data.json`.
Сервер при каждом запросе главной читает данные и подставляет их в шаблон.
